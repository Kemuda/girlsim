# GirlSim 开发计划 —— 基于 Will Wright 游戏设计框架

> Will Wright 是《模拟人生》《模拟城市》《孢子》的创作者。
> 他的 MasterClass 把游戏开发分为四个阶段，强调系统思维、涌现性设计和「玩家作为作者」。
> 本文档用这个框架来组织 GirlSim 的开发路径。

---

## Wright 的四阶段模型

| 阶段 | Wright 的定义 | GirlSim 状态 |
|------|-------------|-------------|
| **Phase 1：研究与灵感** | 深入研究领域，找到有趣的动态系统 | 已完成 |
| **Phase 2：原型与核心机制发现** | 用最粗糙的方式验证核心玩法是否有趣 | 已完成 |
| **Phase 3：系统设计与扩展** | 在核心机制上叠加系统，寻找涌现性交互 | ← 当前阶段 |
| **Phase 4：打磨与生产** | 美术、音效、UI、新手引导 | 未开始 |

---

## Phase 1：研究与灵感 ✓

Wright 的原则：**深入研究，大胆抽象。**

> 他为了做《模拟城市》读了几十本城市规划的书。但关键不是还原现实，而是找到本质动态，丢掉其余的。模拟不需要准确，需要有趣。

我们做了什么：
- 研究了 17 个真实/虚构女性角色的人生轨迹
- 把「一个女性的内在世界」抽象为六个维度：志、幽、韧、渡、身、影
- 把「一生」抽象为 7 个阶段、14 个场景
- 设计了角色宇宙地图（`docs/universes.md`），每个角色有起点→终点面板
- 写了引擎设计文档（`docs/engine-design.md`），包括影子代价和叙事感知系统

### 这个阶段遇到的挑战

**1. 维度命名反复了很多轮**

最初用的是英文直译（Coherence, Depth, Regeneration...），但放在一个讲女性人生的中文游戏里完全没有感觉。试过直译中文（"凝聚力"、"深度"），太干。最后落到六个单字——志幽韧渡身影——每个字背后有古典汉语的意象，但这个过程不是一步到位的，是在不断否定中磨出来的。

Wright 怎么说：「抽象不是简化，是找到本质。」六个字就是六次提炼。

**2. 角色选择是最纠结的部分**

17 个角色不是一开始就有的。过程中：
- 删掉了 4 个不够熟悉的角色（Malala、Maathai、Menchú、Bhutto）——不了解就写不好
- 昂山素季加了又删——现实争议太大，无法给她一个诚实的面板
- 发现「身」维度的角色全是白人——紧急加了邓亚萍
- 专门去找「不优化幽（Depth）但仍然很牛」的女性——因为幽太容易被偏好，需要反例（最终加了 Zaha Hadid、Serena、Coco Chanel）

核心矛盾：**多样性 vs 真实性。** 为了平衡维度分布去硬凑角色，还是只选自己真正理解的人？最终选了后者——宁可维度分布不完美，也不要写一个自己不懂的角色。

**3. 维度平衡：要不要强行拉平？**

统计完 17 个角色后发现，幽和影出现频率明显偏高。要不要为了平衡去调整面板？

结论是不要。这个游戏本身就关注内在世界，幽和影偏多是游戏气质的自然体现，不是设计缺陷。待开发角色（Zaha、Serena 等低幽角色）上线后会自然平衡。

Wright 怎么说：「研究要深入，但抽象要大胆。」大胆的一部分是接受不完美。

**4. 面板只有起点，没有终点**

最初只设计了天赋起点面板。后来才意识到：如果不写终点，就无法验证「这个角色的人生弧线在六维系统里是否讲得通」。补终点面板的过程反过来暴露了起点的问题——有些角色的起点数值需要微调才能让弧线合理。

## Phase 2：原型与核心机制发现 ✓

Wright 的原则：**先做玩具，再做游戏。原型丑没关系，关键是「找到乐趣」。**

> 《模拟城市》最初只是一个城市编辑器。Wright 发现光是看模拟系统的反应就很有趣。玩具先于游戏。

我们做了什么：
- MVP 已上线，完整流程可玩
- 核心循环验证通过：阅读场景 → 做出选择 → 维度变化 → 推进人生 → 抵达结局
- 命运卡机制（40% 概率触发）带来随机性和惊喜

### 这个阶段遇到的挑战

**1. 内容和代码最初是耦合的**

MVP 第一版把所有叙事文本写在组件和逻辑文件里。改一句台词要翻三个文件。后来做了一次大重构，把所有文本抽到 `src/content/`，才实现了内容/引擎分离。

教训：如果你想让非程序员参与共创，架构上必须一开始就为此设计。这个重构早做比晚做好。

**2. 玩家偏离角色天性时，怎么处理？**

这是一个核心设计争论。第一个方案是「引力」机制——偏离角色天性的选择会受到惩罚（数值回拉）。

被否掉了。原因：**惩罚玩家的选择自由，违背了「玩家作为作者」的原则。** 如果玩家想让武则天变成一个温柔的人，那是她的故事。

最终方案：影子代价 + 叙事感知。不惩罚，但让游戏「看见」你在偏离——通过叙事语调的变化来回应，而不是通过数值惩罚。代价是隐性的（影值悄悄涨），体验是叙事的（旁白风格会变）。

Wright 怎么说：「失败应该有趣。」偏离天性不是失败，是一条不同的路，应该通往不同的风景。

**3. 虚构角色的人生是不完整的**

做终点面板时才发现：赫敏在哈利波特结尾才 37 岁，千寻才 10 岁，Elsa 才 24 岁。游戏要玩到角色死亡（65+），但这些角色的原作根本没走完一生。

这个问题还没有解决（见 Issue #3）。可能的方向：标注哪些是「已知」、哪些是「续写」，或者增加一个中间态面板。

**4. TypeScript 的 `verbatimModuleSyntax` 坑**

技术上的小坑但浪费了不少时间：项目开了 `verbatimModuleSyntax`，所有类型导入必须用 `import type {}`，否则 build 直接报错。重构 content 层的时候几乎每个文件都踩了一遍。现在写进了 CLAUDE.md 防止再踩。

### 进入下一阶段前要回答的问题

- [ ] 核心循环有没有乐趣？需要找 3-5 个人试玩观察
- [ ] 选择是否有重量感？还是玩家在随便点？
- [ ] 六维变化是否可感知？玩家是否在乎自己的数值？

Wright 强调：**观察玩家行为，不要访谈。** 看他们玩的时候在哪里停顿、在哪里加速、在哪里笑、在哪里皱眉。

## Phase 3：系统设计与扩展 ← 当前阶段

Wright 的原则：**在核心机制上叠加系统，寻找系统之间的涌现性交互。**

> 简单规则的交互产生复杂行为。设计规则，不要设计结果。如果你能预测每一个结果，你的系统就不够复杂。

这个阶段要做三件事：

### 3a. 填充叙事内容

| 任务 | 说明 | 对应 Wright 原则 |
|------|------|-----------------|
| 为 17 个角色写专属场景线 | 不同角色在同一个人生阶段面对不同的选择 | **可能性空间**——扩大玩家体验的总空间 |
| 打磨选项文案 | 每个选项应该有情感重量，不只是数值调整 | **玩家作为作者**——选项是玩家创作故事的工具 |
| 补全虚构角色的完整人生弧线 | 赫敏/千寻/Elsa 等角色的终局需要续写（见 Issue #3） | **研究深入，抽象大胆** |
| 设计更多命运卡 | 目前 6 张，目标 12-18 张，覆盖更多人生意外 | **失败应该有趣**——命运卡里的「坏事」应该打开新可能 |

### 3b. 实现引擎机制

| 机制 | 说明 | Wright 原则 |
|------|------|------------|
| **角色选择系统** | 游戏开始时选角色，初始六维由角色面板决定 | **玩家投入感**——选择角色 = 选择起点 = 产生归属感 |
| **影子代价** | 偏离角色天性的选择暗中增加影值 | **涌现性**——两个系统（选择系统 + 影值系统）交互产生意外 |
| **叙事感知系统** | 4 级偏离度触发不同叙事风格 | **玩家心理模型**——玩家逐渐意识到「游戏知道我在做什么」 |
| **折线图** | 结局画面展示六维在 7 个阶段的变化轨迹 | **玩家作为作者**——折线图是玩家创作的「一生的形状」 |

### 3c. 持续测试

Wright 说：**每加一个系统就测一次。** 不要等所有系统都做完再测。

- 影子代价做完 → 测：玩家是否意识到影值在涨？
- 叙事感知做完 → 测：玩家是否感受到叙事语调的变化？
- 角色系统做完 → 测：不同角色的体验是否真的不同？

## Phase 4：打磨与生产（未来）

Wright 的原则：**这个阶段花大部分预算，但最少的设计创新。先确保 Phase 3 的系统足够好。**

| 任务 | 说明 |
|------|------|
| UI/视觉风格 | 统一的视觉语言，传达游戏基调 |
| 新手引导 | 让玩家自然地理解六维系统，而不是丢一堆数字 |
| 音效/音乐 | 不同人生阶段的氛围 |
| 部署上线 | 发布到可直接访问的地址（不需要本地运行） |

## Wright 的核心原则在 GirlSim 中的映射

| Wright 原则 | GirlSim 中的体现 |
|-------------|-----------------|
| **玩家作为作者** | 玩家不是在「玩别人的故事」，而是在用选择书写一个女性的一生 |
| **可能性空间** | 17 个角色 × 14 个场景 × 3 个选项 × 命运卡 = 庞大的人生可能性 |
| **涌现性** | 影子代价 + 叙事感知 → 玩家的选择模式本身产生新的叙事层 |
| **失败应该有趣** | 没有「坏结局」——影的结局（暗流）和韧的结局（春草）一样值得抵达 |
| **深入研究，大胆抽象** | 17 个真实女性的完整人生 → 六个汉字 + 1-5 的面板 |
| **先做玩具再做游戏** | MVP 就是那个「玩具」——先跑通核心循环，再加系统 |
| **玩家心理模型** | 六维系统足够直观让玩家建立模型，又足够深让模型持续演化 |
| **观察而非访谈** | 测试时看玩家行为，不问「你觉得怎么样」 |

---

## 一句话总结

Wright 的哲学是：**游戏是帮助人理解复杂系统的最强媒介。**

GirlSim 要让玩家理解的那个复杂系统，是一个女性的内在世界。
