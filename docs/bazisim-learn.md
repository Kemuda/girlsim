# BaZiSim — 学习模式：产品逻辑

> 不是玩命运，是**读**命运。
> 一个用虚拟命局来学习八字的工具。

这是 BaZiSim 的一条分叉线。游戏模式让你「体验」一个命运，学习模式让你「理解」一个命运。

---

## 用户是谁

一个正在学八字的人。她：
- 知道天干地支、十神、身强身弱的基本概念
- 但还不太会「读盘」——看到一个命局，不知道从哪里下手
- 需要大量的练习：看盘 → 推断 → 验证
- 身边没有足够多的真实案例来练手

**核心需求：给我一个命局，让我练习读它，然后告诉我读得对不对。**

---

## 一次使用是什么体验

### 第一步：出题

系统生成一个虚拟命局（或者用户输入一个真实命局）。

```
        年柱      月柱      日柱      时柱
天干    甲(木)    壬(水)    丁(火)    庚(金)
地支    寅(木)    子(水)    亥(水)    申(金)
```

屏幕上只有这八个字。没有任何标注。

### 第二步：你来读

系统引导你一步步分析这个盘，每一步都是一个问题：

**第一关：基础识别**

> Q1：日主是什么？
> → 丁火 ✓
>
> Q2：丁火的阴阳？
> → 阴 ✓
>
> Q3：月令是什么？月令对日主是什么关系？
> → 月令子水，水克火，日主失令 ✓

**第二关：排十神**

> Q4：甲木对丁火来说是什么十神？
> → 甲是阳木，丁是阴火，木生火，异阴阳 → 正印 ✓
>
> Q5：壬水对丁火来说是什么十神？
> → 壬是阳水，丁是阴火，水克火，异阴阳 → 正官 ✓
>
> Q6：庚金对丁火来说是什么十神？
> → 庚是阳金，丁是阴火，火克金……等等，是金克火还是火克金？
> → 火克金。庚金是丁火克的对象，异阴阳 → 正财 ✓

（用户答错时，系统不是直接给答案，而是给提示：「火和金的关系是什么？谁克谁？」）

**第三关：身强弱判断**

> Q7：这个丁火身强还是身弱？说说你的理由。
>
> （用户自由输入）
> "身弱。月令子水失令，日坐亥水也是水，盘里金水太多。年柱甲木可以生火但距离远。"
>
> → 系统反馈：
> ✓ 正确，身弱。
> 你提到了：得令 ✓（失令）、得地 ✓（坐亥，胎位）、得生 △（提到甲木但没分析强度）
> 补充：甲木坐寅，木有根有气，是这个盘里唯一有力的生火来源。但在年柱，离日主远，力量打折。

**第四关：读月柱主题**

> Q8：月柱地支子的藏干是什么？对应什么十神？
> → 子藏癸水，癸是阴水，丁是阴火，水克火，同阴阳 → 七杀 ✓
>
> Q9：所以这个人一辈子的主基调是什么？
> → 七杀——一辈子在跟压力搏斗 ✓

**第五关：看透出**

> Q10：这个盘有没有十神透出？哪些？
> （用户分析藏干和天干的对应关系）

**第六关：推人生**

> Q11：基于以上分析，你觉得这个人的前半生大概是什么样的？
>
> （开放题。用户自由发挥。）
>
> 系统给出参考解读，和用户的答案对比，标出吻合点和遗漏点。

### 第三步：看故事

分析完之后，系统展开这个命局对应的**虚拟人生故事**。

这个故事不是随机的——它是严格按照命局推演出来的。每一个情节转折都对应一个八字概念：

> **童年（大运：偏印）**
> 她六岁的时候，隔壁有个怪老太太每天教她认字。
> ← *偏印：非主流的滋养。身弱得偏印，是保护。*
>
> **青春期（大运：七杀 + 月柱七杀叠加）**
> 学校里最凶的人盯上了她。那一年特别难熬。
> ← *七杀叠加：压力在这个阶段加倍。身弱逢七杀，是考验。*
>
> **中年（大运：正财）**
> 她终于有了自己的收入，但身弱的火守不住财……
> ← *身弱逢正财：机会在眼前但力量不够。*

每个情节旁边都有**八字批注**——像课本的页边注释一样，告诉你：这段故事对应的是什么概念、为什么会这样推。

### 第四步：反复练

换一个盘，再来一次。

系统可以按难度出题：
- **初级**：五行分布明显，身强弱一眼看出
- **中级**：有合局、有冲，需要考虑地支互动
- **高级**：偏枯极端，多个十神透出，藏干复杂
- **实战**：用户输入真实命局，系统辅助分析

## 不同十神作为月柱主题时的学习重点

每个月柱主题对应不同的「人生叙事模式」，也是不同的学习切入点：

| 月柱十神 | 学习重点 | 虚拟人生的特征 |
|---------|---------|-------------|
| **比肩** | 理解「同类」的双面性——帮助 vs 竞争 | 身边总有和她类似的人，是镜子也是对手 |
| **劫财** | 理解身强弱如何翻转同一个十神的含义 | 身强被抢、身弱借力——同一个字，两种命 |
| **食神** | 理解「我生」的温和输出 | 一辈子在创造和养育 |
| **伤官** | 理解叛逆和才华的代价 | 最有戏剧性的人生——大起大落 |
| **偏财** | 理解流动性财富和冒险 | 机会多但不稳定，人生节奏快 |
| **正财** | 理解稳定和积累的逻辑 | 线性成长，但可能被「稳定」困住 |
| **七杀** | 理解压力的双面性——毁灭 vs 淬炼 | 最苦但可能最强的人生 |
| **正官** | 理解规则和体制的力量 | 体面但可能活在别人的期待里 |
| **偏印** | 理解非主流智慧 | 孤独但有洞见 |
| **正印** | 理解传统保护的利弊 | 被照顾得很好，但也被框住 |

系统可以按十神分类出题：「今天我想练习读七杀月柱的盘」→ 系统生成一个七杀月柱的命局。

## 大运流年的学习

这是学八字最难的部分——命局是静态的，但大运让它动起来。

学习模式里，大运不是「自动播放」的，而是**逐步展开**的：

> 系统：这个人的第三步大运是「丙寅」。
> Q：丙火对丁火是什么？寅木对这个盘有什么影响？这步大运对她意味着什么？
>
> 用户分析后，系统展开这十年的故事，验证用户的推断。

每一步大运都是一道题。你先推断，再看答案。七步大运 = 七道题。做完一整套，你就完整地读了一个人的一生。

## 产品形态

| 模块 | 功能 | 用户价值 |
|------|------|---------|
| **出题器** | 生成虚拟命局（可按难度/十神分类） | 无限的练习素材 |
| **读盘引导** | 分步骤的问答式分析（日主→十神→身强弱→月柱→透出） | 学会「怎么读一个盘」 |
| **故事引擎** | 基于命局严格推演的虚拟人生 | 看到理论变成具体的人和事 |
| **批注系统** | 故事的每个情节旁边标注对应的八字概念 | 在故事中理解理论 |
| **大运推演** | 逐步展开大运，先推断后验证 | 练习动态读盘 |
| **自由输入** | 用户输入真实命局，系统辅助分析 | 从练习到实战 |

## 和游戏模式的关系

| | 游戏模式 | 学习模式 |
|--|---------|---------|
| 用户 | 想体验命运的人 | 想理解命运的人 |
| 八字知识 | 不需要懂 | 正在学 |
| 交互方式 | 做选择，体验后果 | 读盘，推断，验证 |
| 八字术语 | 隐藏在后台 | 显性，是学习内容 |
| 故事 | 用户的选择创造的 | 命局推演出来的（不依赖用户选择） |
| 核心体验 | 「如果我是这个命，我会怎么活」 | 「这个命会怎么活，为什么」 |
| 复用的基础设施 | 命局生成、十神计算、大运排列、故事引擎 | 同左 |

两个模式共享底层引擎（八字计算 + 故事生成），但上层交互完全不同。

## 开放问题

- [ ] 故事引擎：虚拟人生的故事是预写的（每种月柱十神对应一套模板）还是 AI 生成的？
- [ ] 批注的颗粒度：每一句话都标注太密，每一段标注又可能遗漏。什么粒度合适？
- [ ] 用户输入真实命局时的责任边界：是否需要免责声明？
- [ ] 社区功能：用户能不能分享自己的读盘分析，让其他学习者点评？
- [ ] 和老师的互动：能不能让八字老师录入他们的解读作为「参考答案」？
