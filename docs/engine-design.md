# Engine Design — 游戏引擎设计文档

> 记录游戏核心机制的设计决策，供开发时参考。

---

## 六维体系

| 维度 | 英文 | 含义 |
|------|------|------|
| **志** | Coherence | 心之所之——方向感与意志力 |
| **幽** | Depth | 隐而深——内在世界的丰富 |
| **韧** | Regeneration | 柔而固——跌倒后站起来的力量 |
| **渡** | Transmission | 济人渡己——对他人的影响与联结 |
| **身** | Body | 安身立命——扎根在肉身里的智慧 |
| **影** | Shadow | 有光才有影——被压抑的、未说出口的部分 |

每个维度取值 0-100。影不是"坏"维度，是"你不敢直视的真实"。

## 角色面板系统

每个角色有三层数据：

```
初始面板（天赋起点）──── 过程折线 ────终局面板（她最终成为的人）
     ↑                    ↑                    ↑
   角色设定            玩家选择累积           结局判定依据
```

### 初始面板

用 1-5 分表示角色的天赋倾向，映射为游戏内初始值：

| 面板分 | 游戏初始值 | 含义 |
|--------|-----------|------|
| 1 | 20 | 极弱——这不是她的领域 |
| 2 | 35 | 偏弱——需要努力才能发展 |
| 3 | 50 | 中等——普通起点 |
| 4 | 65 | 偏强——天赋所在 |
| 5 | 80 | 极强——命运级的天赋 |

影的初始值单独处理：面板分 1=5, 2=15, 3=25, 4=40, 5=55。影的初始值整体偏低，因为影是需要被"积累"的。

### 终局面板

不是预设的，而是玩家选择的结果。结局页展示：

1. **终局六维数值**——柱状图或雷达图
2. **过程折线图**——每个维度在 7 个人生阶段中的变化曲线
3. **与"史实"的对比**（可选）——你的赫敏 vs 原著赫敏的轨迹差异

## 偏离天赋的处理

### 核心哲学

**没有正确答案。** 玩家可以让赫敏变成一个低志高影的人，这是允许的。但偏离天赋应该有**叙事反馈**和**机制代价**，不是惩罚，而是真实——压抑天性确实有代价。

### 机制层：影的代价（Shadow Cost）

当玩家的选择与角色天赋严重不符时，影会额外增长。

规则：
- 计算选择的 delta 方向与角色初始面板的"对齐度"
- 如果选择降低了角色天赋最高的 2 个维度，影 +2（隐性，不显示给玩家）
- 如果选择提升了角色天赋最低的维度且降低了最高的维度，影 +3
- 不是禁止，只是真实：你在压抑自己

示例：
> 赫敏初始面板：志=4, 渡=4（最高），身=2（最低）
> 如果玩家选了一个 志-5 的选项 → 影额外 +2
> 如果玩家选了一个 身+8, 志-3 的选项 → 影额外 +3

### 表现层：叙事感知（Narrative Awareness）

旁白系统感知玩家选择与角色天赋的偏离程度，生成不同风格的文本：

| 偏离程度 | 旁白风格 | 示例 |
|---------|---------|------|
| **顺天赋** | 确认、流畅 | "你做了你一直会做的选择。水流入了它本该去的河道。" |
| **轻微偏离** | 好奇、新鲜 | "这不太像你。但也许正因如此，它让你觉得活着。" |
| **显著偏离** | 内心冲突 | "你选了一条陌生的路。心里有什么在抗议，但你没有停下来。" |
| **极端偏离** | 身份危机 | "你已经不认识镜子里的自己了。这是自由，还是迷失？" |

偏离度计算：
- 取选择 delta 中变化最大的维度
- 与角色初始面板对比
- 如果该维度在初始面板中是最低的 2 个，且变化为正向 → 轻微偏离
- 如果选择同时降低了初始最高维度 → 显著偏离
- 如果累计偏离（多次逆天赋选择）达到阈值 → 极端偏离

### 被否决的方案：引力机制

曾考虑让天赋产生"引力"——顺天赋选 delta 加成，逆天赋选 delta 削弱。

**否决原因**：这本质上是在惩罚玩家的自由选择，与"没有正确答案"的设计哲学冲突。我们希望玩家可以自由探索，只是偏离会有**叙事后果**和**影的积累**，不是数值惩罚。

## 选择系统

### 每个选择的数据结构

```typescript
interface Choice {
  text: string;                        // 选项文字
  delta: Partial<CharacterState>;      // 六维变化
  // 未来扩展：
  // narrativeTag?: string;            // 用于旁白系统的标签
  // requiredMin?: Partial<CharacterState>;  // 前置条件（可选）
}
```

### Delta 设计原则

1. **每个选择影响 2-3 个维度**，不超过 4 个
2. **正向变化通常 +3 到 +10**，负向变化 -3 到 -5
3. **影不直接出现在大多数选项的 delta 中**——影主要通过"偏离天赋"机制自然积累
4. **没有选择是纯正面或纯负面的**——每个选择都是取舍

## 阈值事件（命运卡）

### 触发条件

- 在人生阶段转换时有概率触发（当前设定 40%）
- 每张卡只能触发一次
- 四种类型有不同的叙事功能：

| 类型 | 功能 | 对六维的典型影响 |
|------|------|----------------|
| **Disruption（打断）** | 打破现有平衡 | 高 delta，有正有负 |
| **Opening（契机）** | 提供新的可能性 | 中等正向 delta，影可能降低 |
| **Mirror（镜像）** | 照见真实的自己 | 影响幽和影 |
| **Ghost（幽灵）** | 与"未活出的自己"对话 | 影响幽和韧 |

### 未来考虑

- 阈值事件可以根据当前六维状态动态选择（不纯随机）
- 当影超过某个阈值时，Ghost 类事件概率增加
- 当某个维度极端高时，Mirror 类事件概率增加

## 结局判定

### 当前逻辑

找到最高维度，匹配对应结局。如果最高维度达不到阈值，fallback 到"河流"结局。

### 未来改进方向

- 考虑面板的"形状"而非单一最高值——均衡型、极端型、双高型 应该有不同结局
- 考虑过程而非只看终值——一个一直高志然后最后崩溃的人，和一个一直低志的人，故事不同
- 影的角色：影不只是"高=坏结局"，高影+高韧 = 一种独特的力量

## 折线图（过程可视化）

### 数据采集

每次选择后记录当前六维状态，形成时间序列：

```typescript
interface Snapshot {
  turnIndex: number;      // 人生阶段 0-6
  sceneId: string;        // 场景 ID
  state: CharacterState;  // 六维快照
}
```

### 展示

结局页展示折线图：
- X 轴：7 个人生阶段（童年 → 暮年）
- Y 轴：0-100
- 6 条线，每条一个维度，用维度对应的颜色
- 可选：叠加显示"史实轨迹"的虚线作为对比
